# Дизайн-спецификация: Радикальная очистка кода и настройка ruff

- Дата: 2026-05-06;
- Статус: Утверждено;
- Цель: Удаление всех комментариев и строк документации из исходного кода с последующей автоматической нормализацией через ruff.

## 1. Метод очистки кода
Для удаления комментариев и докстрингов будет использован Python-скрипт на основе модуля `ast` или `tokenize`.
- удаление всех строк документации (функций, классов, модулей);
- удаление всех комментариев, начинающихся с `#` (кроме shebang, если они есть);
- сохранение структуры кода и логики выполнения.

## 2. Настройка линтера ruff
После очистки кода будет настроен `ruff` для поддержания чистоты и единообразия:
- конфигурация в `pyproject.toml`;
- включение правил удаления неиспользуемых импортов и переменных (F401, F841);
- форматирование кода (ruff format).

## 3. Риски
- полная потеря текстовых пояснений к сложной математической логике или параметрам моделей;
- возможное нарушение форматирования, которое будет исправлено на этапе запуска `ruff`.
