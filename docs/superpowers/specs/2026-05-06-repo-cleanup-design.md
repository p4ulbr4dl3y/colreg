# Дизайн-спецификация: Очистка и организация репозитория COLREG Video Analytics

- Дата: 2026-05-06;
- Статус: Утверждено;
- Цель: Приведение структуры репозитория в порядок, удаление временных файлов и логическая группировка ресурсов.

## 1. Изменения в структуре файлов

### 1.1. Удаление файлов
- `refactor_imports.py`: скрипт для разового рефакторинга импортов, более не актуален;
- `.DS_Store`: системные файлы macOS;
- `requirements.txt`: после переноса зависимостей в `pyproject.toml`.

### 1.2. Перемещение файлов
- `datasets.md` -> `/Users/yegor/datasets.md`: вывод информации о датасетах за пределы проекта по запросу пользователя;
- `debug_perf.py` -> `tools/debug_perf.py`: скрипт для замера производительности;
- `amqtt.yaml` -> `scripts/amqtt.yaml`: конфигурация MQTT брокера рядом со скриптами, которые её используют;
- `test_day_images.py` -> `tests/integration/test_day_images.py`: интеграционный тест для дневного режима;
- `test_night_images.py` -> `tests/integration/test_night_images.py`: интеграционный тест для ночного режима.

## 2. Управление зависимостями
- все зависимости из `requirements.txt` должны быть перенесены в секцию `dependencies` файла `pyproject.toml`;
- формат записи должен соответствовать стандартам `PEP 621` и поддерживаться `uv`.

## 3. Новые директории
- `tools/`: для вспомогательных скриптов разработки и профилирования;
- `tests/integration/`: для тяжелых интеграционных тестов, требующих внешних данных (изображений).

## 4. Сохраняемая структура
- `src/`: исходный код проекта;
- `models/`: веса обученных моделей;
- `test_images/`: данные для тестов;
- `scripts/`: основные рабочие скрипты (node, simulator).
